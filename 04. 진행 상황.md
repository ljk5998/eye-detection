# 04. 진행 상황

## 1. 현재 프로젝트 상태

본 프로젝트는 **홍채 중심(Iris Center) 기반 안구 운동 스크리닝 도구**로 방향을 확정했습니다.

초기 딥러닝 분류 모델 방식이 아니라, MediaPipe로 좌/우 홍채 중심 좌표를 추적하고 프레임 간 이동량과 속도를 계산해 안구 운동 패턴을 기록하는 방식입니다.

현재 GitHub 저장소:

```text
https://github.com/ljk5998/eye-detection
```

현재 원격 브랜치:

```text
main -> origin/main
```

초기 업로드 커밋:

```text
b46b0f1 Initial iris screening prototype
```

## 2. 확정된 방향

### 기준점

*   동공(Pupil)이 아니라 **홍채(Iris)** 기준으로 진행합니다.
*   좌/우 홍채 외곽 랜드마크의 평균 좌표를 홍채 중심으로 사용합니다.
*   좌표는 원본 픽셀 좌표와 눈 구석 기준 상대 좌표를 함께 저장합니다.

### 프로젝트 표현

*   본 도구는 의료적 진단 도구가 아닙니다.
*   문서 표현은 **진단**, **판정**보다는 **스크리닝**, **관찰 지표**, **패턴 기록** 중심으로 정리했습니다.
*   이상 탐지 임계값은 아직 확정하지 않았고, 먼저 데이터를 수집한 뒤 기준을 정하기로 했습니다.

### 1차 프로토타입 목표

1.  웹캠 또는 영상 파일 입력
2.  MediaPipe 기반 얼굴/홍채 landmark 추출
3.  좌/우 홍채 중심 좌표 계산
4.  프레임 간 이동량 `dx`, `dy` 계산
5.  홍채 중심 이동 속도 계산
6.  실시간 오버레이 표시
7.  CSV 로그 저장

## 4. 프로토타입 코드 요약

메인 실행 파일:

```text
src/iris_screening/prototype.py
```

주요 기능:

*   `--source 0`: 0번 카메라 사용. 일반적으로 Mac 내장 웹캠입니다.
*   `--source data/raw/sample.mp4`: 영상 파일 분석.
*   `--no-preview`: 화면 표시 없이 CSV만 저장.
*   `--output`: CSV 저장 경로 지정.
*   `--model`: MediaPipe Tasks API용 `face_landmarker.task` 모델 경로 지정.

기본 실행:

```bash
python src/iris_screening/prototype.py --source 0
```

CSV만 저장:

```bash
python src/iris_screening/prototype.py --source 0 --no-preview
```

기본 출력 파일:

```text
data/processed/iris_trace.csv
```

## 6. MediaPipe 관련 메모

현재 로컬 Anaconda Python 환경에서는 다음 상황이 확인되었습니다.

```text
Python: /opt/anaconda3/bin/python
MediaPipe: 0.10.33
mp.solutions: 없음
mp.tasks: 있음
```

따라서 기존 예제에서 자주 쓰는 아래 방식은 현재 환경에서 바로 동작하지 않습니다.

```python
mp.solutions.face_mesh.FaceMesh(...)
```

이를 보완하기 위해 `prototype.py`는 두 가지 방식을 모두 지원합니다.

1.  `mp.solutions.face_mesh`가 있으면 legacy Face Mesh API 사용.
2.  `mp.solutions`가 없으면 MediaPipe Tasks API의 `FaceLandmarker` 사용.

Tasks API에서는 모델 파일이 필요합니다.

```text
data/models/face_landmarker.task
```

해당 모델 파일은 크기가 있고 재다운로드 가능하므로 Git에는 올리지 않습니다. `.gitignore`에 의해 제외됩니다.

## 9. 알려진 이슈

### macOS 카메라 권한

처음 웹캠을 실행하면 macOS에서 Terminal 또는 Python 카메라 권한을 요구할 수 있습니다.

권한 문제가 발생하면 다음 위치에서 허용합니다.

```text
System Settings > Privacy & Security > Camera
```

### MediaPipe Tasks 그래픽 서비스 이슈

에이전트 실행 환경에서는 MediaPipe Tasks detector 생성 시 다음 계열의 오류가 발생할 수 있었습니다.

```text
kGpuService
NSOpenGLPixelFormat
```

이 경우 일반 Terminal 세션에서 실행하거나, Python 3.12 환경에서 legacy `mp.solutions.face_mesh`가 지원되는 MediaPipe 조합을 사용하는 방향을 검토합니다.

### 이상 탐지 기준 미정

현재는 데이터 수집용 프로토타입입니다. 아래 기준은 아직 확정하지 않았습니다.

*   baseline으로 사용할 초기 프레임 수
*   정상 좌표 범위
*   속도 임계값
*   Z-score 기준
*   moving average window
*   anomaly score 계산식
